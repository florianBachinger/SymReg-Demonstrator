GP Settings

Allowed Symbols
Choose which mathematical operations the GP algorithm can use to build expressions. More symbols = more expressive but slower search. constant and variable are always required. Select function primitives
Primary
Number of evolutionary generations. Higher = more time to evolve better solutions but slower. Lower = faster but may find simpler/worse fits. Typical range: 10–100. generations
Number of candidate solutions in each generation. Higher = more diversity, better exploration, but more memory and time. Lower = faster iterations but risk premature convergence. Typical: 100–1000. population_size
Maximum tree length (number of nodes) for candidate expressions. Higher = allows more complex expressions. Lower = forces simpler, more interpretable models. Typical: 10–50. max_length
Secondary
Probability of mutating an individual each generation (0–1). Higher = more exploration, can escape local optima. Lower = more exploitation of good solutions. Typical: 0.05–0.3. mutation_probability
Number of individuals competing in tournament selection. Higher = stronger selection pressure, favors best individuals. Lower = weaker pressure, more diversity. Typical: 2–7. tournament_size
Number of local optimization iterations (gradient-based) applied to each individual's constants. Higher = better-tuned coefficients but slower. Lower = faster but coefficients may be rough. Typical: 5–50. optimizer_iterations
Maximum number of fitness evaluations before stopping. Acts as a computational budget. Higher = more thorough search. Lower = strict time limit. Set to 0 for unlimited. Typical: 10,000–1,000,000. max_evaluations
Advanced
Criterion used to select the best model balancing fit quality vs. complexity. MDL = minimum description length (Occam's razor). BIC/AIC = Bayesian/Akaike information criterion. MDL tends to prefer simpler models. model_selection_criterion
Local optimization method for tuning constants. lm = Levenberg-Marquardt (fast, good for nonlinear least squares). Generally leave as default. optimizer
Size of the offspring pool generated each generation. Usually set equal to population_size. Higher = more offspring to evaluate. Typical: same as population_size. pool_size
Convergence threshold for the optimizer. Smaller values allow finer optimization but may take longer. Typical: 1e-3 to 1e-6. epsilon
Seed for the random number generator. Set a number for reproducible results. Leave blank for random initialization each run. random_state
Assumed measurement uncertainty of the data. Used in model selection criteria. Higher = more tolerance for imprecise fits, favors simpler models. Typical: 0.01–0.1. uncertainty
Number of threads for parallel evaluation. 0 = auto-detect (use all cores). Set to 1 for single-threaded execution. Higher values speed up large populations. n_threads
